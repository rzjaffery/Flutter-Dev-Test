# Flutter Product Management App

A full-featured Flutter application with **Firebase Auth** and **Firestore** covering user authentication, product CRUD, paginated grid view, and more.

---

## Features

| Feature | Details |
|---|---|
| **Register** | Create account with name, email, password |
| **Login** | Email + password sign-in |
| **Forgot Password** | Firebase sends a reset link to your email |
| **Change Password** | Update password while logged in (re-authenticates) |
| **Create Product** | Name, description, price, category, optional image URL |
| **Grid View** | 2-column paginated grid (6 per page, infinite scroll) |
| **Product Detail** | Full view on tap |
| **Edit Product** | Pre-filled form, updates Firestore |
| **Delete Product** | Confirmation dialog before deletion |
| **Logout** | Signs out with confirmation dialog |

## Setup Guide

### Step 1 — Prerequisites

Make sure you have:
- Flutter SDK ≥ 3.0 installed
- A Google account
- Firebase CLI installed: `npm install -g firebase-tools`
- Node.js ≥ 14

### Step 2 — Create a Firebase Project

1. Go to https://console.firebase.google.com
2. Click **"Add project"** → give it a name → Continue
3. (Optional) Disable Google Analytics → **Create project**

### Step 3 — Enable Firebase Authentication

1. In your Firebase console, go to **Build → Authentication**
2. Click **"Get started"**
3. Under **Sign-in method**, enable **Email/Password** → Save

### Step 4 — Enable Firestore Database

1. Go to **Build → Firestore Database**
2. Click **"Create database"**
3. Choose **"Start in test mode"** (we'll apply proper rules after)
4. Select a Cloud Firestore location → **Enable**

### Step 5 — Deploy Firestore Security Rules

Run from your project root:
```bash
firebase login           # log in if you haven't already
firebase init firestore  # select your project, accept default files
firebase deploy --only firestore:rules
firebase deploy --only firestore:indexes
```

### Step 6 — Connect Flutter to Firebase (FlutterFire CLI)

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# From your Flutter project root:
flutterfire configure
```

- Select your Firebase project from the list
- Choose the platforms you want (Android, iOS, Web, etc.)
- This **generates/overwrites** `lib/firebase_options.dart` with real credentials

### Step 7 — Android Setup (if targeting Android)

1. In Firebase Console → **Project Settings → General**
2. Under "Your apps", add an Android app
3. Set **Android package name** to match your `applicationId` in `android/app/build.gradle`
4. Download `google-services.json` → place it in `android/app/`
5. In `android/app/build.gradle`, add:
   ```gradle
   apply plugin: 'com.google.gms.google-services'
   ```
6. In `android/build.gradle`, add to `buildscript > dependencies`:
   ```gradle
   classpath 'com.google.gms:google-services:4.4.0'
   ```

### Step 8 — iOS Setup (if targeting iOS)

1. In Firebase Console → add an iOS app
2. Set Bundle ID to match `PRODUCT_BUNDLE_IDENTIFIER` in `ios/Runner.xcodeproj`
3. Download `GoogleService-Info.plist`
4. In Xcode, drag it into the `Runner/` folder (check "Copy items if needed")
5. Minimum iOS version must be ≥ 13 — set in Xcode project settings

### Step 9 — Install Packages & Run

```bash
flutter pub get
flutter run
```

## Firestore Data Model

### `/users/{uid}`
```json
{
  "uid": "abc123",
  "email": "user@example.com",
  "displayName": "John Doe",
  "createdAt": "2025-01-01T00:00:00.000Z"
}
```

### `/products/{productId}`
```json
{
  "userId": "abc123",
  "name": "Wireless Headphones",
  "description": "High-quality sound...",
  "price": 49.99,
  "category": "Electronics",
  "imageUrl": "https://example.com/image.jpg",
  "createdAt": "2025-01-01T00:00:00.000Z",
  "updatedAt": "2025-01-01T00:00:00.000Z"
}
```

## Testing the App

1. **Register** a new account
2. **Login** with your credentials
3. **Create** several products (add an image URL like `https://picsum.photos/400/300`)
4. **Scroll** down to trigger pagination (loads 6 at a time)
5. **Tap** a product to view details
6. **Edit** → change price or description → Update
7. **Delete** → confirm via dialog
8. **Forgot Password** → enter email → check inbox
9. **Change Password** → enter current + new password
10. **Logout** → confirms and returns to Login

## Common Issues

| Issue | Fix |
|---|---|
| `firebase_options.dart` throws error | Run `flutterfire configure` |
| Android build fails | Ensure `google-services.json` is in `android/app/` |
| Auth operations fail | Check Email/Password is enabled in Firebase Console |
| Firestore reads fail | Check security rules allow the user's UID |
| Pagination query fails | Deploy `firestore.indexes.json` with `firebase deploy --only firestore:indexes` |
